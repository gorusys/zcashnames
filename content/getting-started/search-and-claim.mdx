# Search & Claim a Name

## Searching for a name

Type any name into the search bar on the homepage. The system checks if it's available, already taken, or listed for sale.

| Status | What it means |
|--------|---------------|
| **Available** | Nobody owns this name yet. You can claim it. |
| **For Sale** | Someone owns it but has listed it on the marketplace. You can buy it. |
| **Registered** | Taken and not for sale. |

Both `.zcash` and `.zec` suffixes point to the same name — they're interchangeable.

## How claiming works

1. Search for a name that shows as **Available**
2. Enter your Zcash address — this is where the name will point
3. The app generates a payment link (a ZIP-321 URI) with the exact amount and a memo
4. Open the link in your Zcash wallet (or scan the QR code) and send the transaction
5. Once the transaction confirms on-chain, the name is yours

That's it. The indexer watches the blockchain, sees your transaction, validates the memo, and registers the name to your address.

## Things to know

- **First-come-first-served:** if two people try to claim the same name, whoever's transaction confirms first wins
- **Use a unified address** (`u1...` or `utest1...`) for the best privacy. Other address types work but leak more information on-chain
- **Shorter names cost more** because they're scarcer — see [Pricing](pricing)
- **Names are permanent** — no renewal fees, but you need to stay active once every 180 days

## Wallet requirements

Your wallet needs to support three things to work with ZNS:

1. **ZIP-321 payment URIs** — to parse the `zcash:` links the app generates
2. **Orchard shielded transactions** — all ZNS operations use the Orchard pool
3. **Memo field** — the wallet must include the memo exactly as provided (do not modify it)
